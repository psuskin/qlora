This is the description of the functionality of the class "CX_PRODUCT_AMOUNT - CyberEnterprise" regarding Code example: Attention: If several values with units are together in a Product Amount, their units are also multiplied! So if there are 2m and another 2m in the Product Amount, the result is 4m2. If this behaviour is not desired, as is the case with the quantity formation of nested production parts lists, a standardisation factor must be integrated between all individual values.
