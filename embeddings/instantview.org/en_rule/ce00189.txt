This is the description of the class "CX_PERIODIC_DATE - Comparisons": Description The comparison operators ">",<" und "=" vergleichen ein periodisches Datum mit einem Datum beliebigen Typs. Dabei gelten folgende Regeln: Zwei periodische Daten gleichen sich, wenn beide genau die gleichen Informationen enthalten. (Es darf also keine Unterschiede bei Startdatum, Häufigkeit, Bezugszeitraum, Position, Tag-Codierung und Einzelperioden-Dauer geben.) Ein periodisches Datum gleicht einer "date period, if they contain exactly the same days. This is only possible if both start on the same day, the periodic date contains all possible days, and the date range is endless, or if the periodic date has no start and includes all possible days, while the date range has neither start nor end. A periodic date is always smaller than another periodic date, unless it is the same (see 1.). A periodic date is always greater than a current date. A periodic date is always greater than a date period (span) if it has an end and a beginning. It is smaller if there is no span start from span end, unless they are equal (see 2.) If there is no beginning of a chip, but an end, it is always bigger. If there is no end, it is smaller unless it has a start value or the start value begins before the date period. The above-mentioned (perhaps seemingly complicated) rules can be reduced to a fairly simple rule of memory. Two date definitions are equal if they contain exactly the same days. One date is always smaller than another if the two dates are not the same and the number of days is smaller.
