This is the description of the functionality of the class "CX_ACCESS_CONTROL" regarding Description: Class of the ACCESS_CONTROL-Manager. There is exactly one user/group object for each SID. Unless set otherwise, the Active Directory server is set to the Windows environment variable "%LOGONSERVER%", which is the one Windows is currently using, at each startup. If the environment variable %LOGONSERVER%contains"\\MicrosoftAccount", you are only logged in locally. If you log on to a computer with a local account and not a domain account, these functions will not work.
