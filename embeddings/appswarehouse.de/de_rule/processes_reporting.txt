Dies ist die Beschreibung des Moduls "processes_reporting" mit dem Namen "Berichte": Einleitung Aufruf Übersicht über alle Standardreports Merkmale im Bericht. Dies ist die Beschreibung der Funktionalität des Moduls "processes_reporting" mit dem Namen "Berichte" bezüglich Einleitung: Bei Berichten bzw. Reports handelt es sich um die Zusammenstellung betrieblicher Information und diese meist in Listendarstellung (z.B. eine Liste aller noch nicht oder nur teilweise gelieferten Aufträge). Sie können sowohl für den innerbetrieblichen oder für den externen Gebrauch erstellt werden. Für die Weitergabe der Informationen kommen verschiedene Medien in Frage, so können Berichte in Papier- oder elektronischer Form erstellt werden. Ein Report unterscheidet sich in der Regel von einem Dokument, das zur Ausgabe eines Beleges verwendet wird. Ein Beispiel ist die Ausgabe einer Auftragsbestätigung, einer Rechnung, usw. per Brief (Papier) oder z.B. auch als PDF-Anhang einer Email. Dies ist die Beschreibung der Funktionalität des Moduls "processes_reporting" mit dem Namen "Berichte" bezüglich Aufruf: ClassiX® bietet verschiedene Auswertungen, wie zum Beispiel die Auftragskostenauswertung, an. Dies ist bereits ein Report, welcher allerdings nicht ohne definierten Bericht ausgegeben werden kann. Für die Ausgabe von Reports ist immer eine Liste notwendig, welche die Datenbasis des Berichtes darstellt. So ist das Ergebnis der Auftragskostenauswertung eine Liste mit allen Informationen. Über einfache Listen, wie zum Beispiel der Auftragsliste in der entsprechenden Selektionsmaske, kann aber ebenfalls ein Bericht erstellt werden. Über diesen Button, der sich in der Regel in der Toolbar befindet, wird das Reporting-Modul aufgerufen. Über die Lasche "Zwischenablage" kann eine Liste, so wie sie im aufrufenden Fenster angezeigt wird, kopiert und anschließend Beispielsweise in eine Textdatei oder Excel eingefügt werden. Über die Lasche "Berichte" können Reports, auf Grundlage einer Datenbasis erstellt und gedruckt werden. Als Datenbasis dient entweder die Liste aus dem aufrufenden Fenster oder eine eigens für den Report definierte, interne Liste. Ebenfalls ist es möglich, dass der Report eine eigene Datenbasis definiert. Ob dies der Fall ist, kann man am Symbol des Berichtes erkennen. 'Normalen' Report. Report mit einer eigenen Datenbasis. Dies ist die Beschreibung der Funktionalität des Moduls "processes_reporting" mit dem Namen "Berichte" bezüglich Merkmale im Bericht: Es können im Bericht Merkmale definiert werden die vom Benutzer vor dem Ausdruck manuell verändert werden können. Diese Mermale werden in dem Dataset "dsVariable" mit ihrem Namen eingefügt und können so im Bericht verwendet werden.
