Dies ist die Beschreibung des Moduls "conditionedConditionEdit" mit dem Namen "Bedingte Konditionen": Zur effizienteren Erfassung von Angeboten/Aufträgen kann man kundenspezifische Konditionen in der Kundenrolle hinterlegen, welche beim Anlegen eines Angebotes/Auftrages automatisch übernommen werden.Da man diese Regel aber nicht auf alle Konditionen verallgemeinern kann, gibt es die Möglichkeit Konditionen mit einer Bedingung zu versehen. Dies ist die Beschreibung der Funktionalität des Moduls "conditionedConditionEdit" mit dem Namen "Bedingte Konditionen" bezüglich Funktionalität: Nach der Auswahl wird die Bedingte Kondition direkt angelegt und in der eigenen Editiermaske angezeigt. In der Liste werden die bedingten Einträge dargestellt. Ein Eintrag besteht aus folgenden Informationen. Als erstes sollte jede Bedingung mit einem sinnvollen Namen versehen werden, damit auch ein anderer Benutzer weiß worum es sich handelt, wenn er die Kondition editiert. Als nächstes kann/sollte die Bedingung definiert werden, welche beim Einfügen der Kondition in einen Auftrag ausgewertet wird. Da das Definieren dieser Bedingung oft weitergehende Kenntnisse als die eines normalen Anwenders erfordert, können über den Button Standardbedingungen voreingestellt Bedingungen übernommen werden. Die Liste der Bedingungen kann auf Anfrage selbstverständlich erweitert werden. Als letztes muss noch eine Kondition hinterlegt (eingefügt/angelegt) werden. Dies geschieht über das Kontextmenü des Konditionsfeldes. Über den Speicherbutton werden die vorgenommenen Änderungen gespeichert. Neue bedingte Einträge können über das Kontextmenü der Liste angelegt und bestehende gelöscht werden. Die Pfeile neben der Liste dienen dem verschieben der bedingten Einträge. Hinweise, Erläuterungen, Beispiele Wird eine bedingte Kondition mit mehreren Einträgen angelegt, so wird hierfür im Auftrag/Angebot minimal keine und maximal eine Kondition übernommen. Sollten mehrere Bedingungen zutreffen, so wird immer die erst Kondition mit einer richtigen Bedingung übernommen. Dadurch ist es auch sinnvoll, dass man die Einträge über die Pfeilbuttons positionieren kann. Hat ein Eintrag keine Bedingung, so gilt die Bedingung immer als wahr. Achtung! Die Bedingung wird jeweils beim Einfügen der Kondition ausgewertet. Sollte also eine Bedingung von der Auftragsart abhängig sein, so sollte dieses bereits gesetzt sein BEVOR der Kunde ausgewählt wird. Aus technischer Sicht ist es derzeit nicht möglich mehrere Konditionen über eine bedingte Kondition in einen Auftrag einzufügen. Statt dessen müsste man mehrere bedingte Konditionen mit gleichen Bedingungen beim Kunden hinterlegen.
