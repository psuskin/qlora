This is the description of the module "qm_deadlock_qm" with the name "Deadlock retry": The deadlock retry mechanism has the following properties: if a deadlock occurs within a transaction, the transaction will the environment parameters are restored (stack data, variables) (see also the test Resuming after err_deadlock) and an attempt is made to carry out the transaction again If the environment variable CX_DEADLOCKRETRY= is set, the output of the deadlock mechanism is stored in the specified file.
