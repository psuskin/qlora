This is the description of the module "qm_resume_qm" with the name "Resuming after err_deadlock": The deadlock retry mechanism has the following properties: if a deadlock occurs within a transaction, the transaction will the environment parameters are restored (stack data, variables) and an attempt is made to execute the transaction again (see also the Test Deadlock retry) If the environment variable CX_DEADLOCKRETRY= is set, the output of the deadlock mechanism is stored in the specified file.
