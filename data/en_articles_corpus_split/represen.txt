represen. Description: Within the CyberEnterprise business OS, the partner function "Representative" represents persons or companies who work for you in sales. Representatives can therefore also be commission recipients, the latter term is used synonymously within the CyberEnterprise business OS. Input window: This module is used to maintain the substitute data. You can select a substitute by entering the substitute number in the Number field and confirming with RETURN. If you want to call up the substitute list, you can either double-click the "Number" field or select the menu item "Search". Fields marked with "italics" represent data directly from the company. If something is changed in one of these fields, the change is applied to all roles and the company itself. All this data can also be maintained directly in the company input window. List window: This window is used to list the representatives. Selection window: This window is used to select a substitute. Option window: All settings for the search can be made here. The window can only be closed with the "OK" button. Module name: repreedt.mod. Classes: CX_BANK_ACCOUNT CX_BRANCHES CX_CORPORATION CX_REPRESENTATIVE CX_PERSON CX_PO_BOX_ADDRESS CX_STREET_ADDRESS CX_STRUCTURE. Security: In addition to restricting access rights via the class and its data fields, the module can be restricted in its use via some of the received messages. Module name: represel.mod. Classes: CX_REPRESENTATIVE.
