cxWorkTimeEvent: Time management events are occurrences that have an influence on the stamped/recorded total working time (flexitime, recording of hours) and/or on an employee's absence within a working day. These are, for example, the usual "coming" and "going" events, as well as the start and end of breaks, doctor's visits, full-day absences due to illness, unpaid private absences etc.Such events are directly assigned when "stamping" the attendance and absence times, when recording the absences of an employee and when storing the working time models. Input window: This window is used to maintain events within time management. You can select a specific event by entering the desired number in the Number field and confirming with RETURN. If you want to call up the list window, you can select the menu item "Search". Events are used for posting clock-in and clock-out messages when employees clock in and out, as well as in the master data of working time models. (See also the description above!). List window: Used to list the events. Selection window: This window is used to select an event. Module name: flexeven.mod. Classes: CX_FLEXTIME_EVENT.
