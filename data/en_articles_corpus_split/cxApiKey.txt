cxApiKey: This module is used to maintain API keys. An API key is required to access an application programming interface (API). In addition, the use of an API key can be restricted to certain source IP addresses. Furthermore, the validity period of the API key can be defined.Logically, an API key is the reverse of a web service that provides access to an external resource. Input window: This window is used to maintain API keys. The window is divided into two tabs. The most important is the first tab "API key". It contains the option to define the API key as a text entry for the selected object. In addition, permitted IP addresses can be stored in the form of a whitelist for the API key. The second tab "Characteristics" offers the possibility to add the characteristics (attributes) defined in ClassiXÂ® to an API key and to assign corresponding values. List window: This window is used to list the API keys. Double-clicking on an API key takes you directly to its input window. Selection window: This window is used to select an API key. Options window: All settings for the search can be made here. Module name: cxApiKey.mod. Classes: CX_API_KEY. Module name: apiKeyEdit.app. Classes: CX_API_KEY. Security: In addition to restricting access rights via the class and its data fields, the module can be restricted in its use via some of the received messages.
